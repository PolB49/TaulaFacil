ERROR 1

Falta de dependències
requires java.sql; requires javafx.swing; requires java.desktop; requires javafx.base;

Dins del document “module-info”, faltaven dependències de java que s'haurien d'haver importat automàticament al modificar el pom per conectar amb la base de dades. Per aluna raó, aqueses dependencies no es van importar i es va haver de fer manualment.
Principalment aquestes dependències provocaven un error que evitava fer els imports necessaris per a la connexió de base de dades.

ERROR 2

Import incorrecte

El TextField no es detectat i no s'inicia l'aplicació

Per solucionar-ho s’ha de fer l’import correcte.

Aquest error va apareixer a l'inici del desenvolupament de l'aplicació. Principalment va estar causat per una distracció a l'hora de seleccionar l'import, ja que es van seleccionar unes dependencies diferents a les necesaries per a JAVA FX

ERROR 3

Classe estàtica

Error del cambrer Actual. Al moment d'iniciar sessió o registrar-se a la app, les dades del cambrer s'haurien d'emmagatzemar en forma de clase "Cambrer". Dins del model, es té una variable anomenada "Cambrer Actual". Aquesta variable retorna una instancia de la classe "Cambrer" que representa el cambrer el qual ha iniciat sessió. Fins que s'arriba al "TerciaryControll" aquesta variable es nula, llavors es quan s'actualitza amb les noves dades. L'error es trobava que si la variable "Camber Actual" no es declarava com a estàtica, les dades s'actualitzaven pero no es guardaven i per tant, segui notificant com que el "Cambrer Actual" era una variable nula.

ERROR 4

ListView no mostrava el preu

Al listview de les comandes no es mostrava el preu unitari, ja que al recopilar el observable list de comandes, no es feia un join a la base de dades entre les taules "comandes" i "producte". Per tant, no es podia recuperar el preu individual de cada producte i es mostrava el preu implementat per defecte és a d, 0.00€
